# 機能一覧

GitHub Copilot Spec-Driven Development の主要機能の詳細ドキュメント

## 🎯 コア機能

### [完全 GitHub 統合](./github-integration.md)

仕様から実装、レビューまでを GitHub エコシステム内で完結。Issues、Pull Requests、Projects、Actions との自動連携。

**主な機能:**

- Issue 自動同期
- PR 自動作成
- GitHub Projects 統合
- CI/CD パイプライン連携

---

### [デバッグ駆動修正 (DDF)](./debug-driven-fixing.md)

AI が「なぜそう修正したか」を明確に説明できる構造化デバッグフロー。根本原因の特定を支援。

**主な機能:**

- 症状の構造化記録
- 根本原因の仮説検証
- 修正の影響範囲分析
- 再発防止の自動提案

---

### [批判的対話](./critical-dialogue.md)

AI が質問ではなく提案形式で応答し、Goal・Constraint・Reference の 3 軸で意図を固定。曖昧性を構造的に排除。

**主な機能:**

- 意図の三角測量
- 提案ベース対話
- 曖昧性の検出と解消
- 決定の記録と追跡

---

### [三層記憶アーキテクチャ](./triple-memory.md)

Constitution（不変の原則）+ Context（可変の記憶）+ Changelog（変更履歴）の統合メモリシステム。

**主な機能:**

- プロジェクト憲法管理
- コンテキスト自動更新
- 変更履歴の追跡
- メモリの一貫性保証

---

### [段階的導入パス](./progressive-adoption.md)

Level 0（ゼロコスト）から Level 3（フル機能）まで、プロジェクトの成熟度に合わせて段階的に導入可能。

**主な機能:**

- レベル別機能セット
- 段階的な学習パス
- 移行支援ツール
- ROI 測定ダッシュボード

---

### [多言語対応](./multilingual.md)

日本語、英語を含む多言語でのプロンプト最適化。各言語の特性に合わせた AI 対応。

**主な機能:**

- 多言語テンプレート
- 言語別プロンプト最適化
- 自動翻訳サポート
- 文化的コンテキストの考慮

---

### [柔軟な開発原則](./flexible-principles.md)

Constitution で自由に開発原則を定義可能。TDD、SOLID、DDD など、プロジェクトに最適な原則を選択。

**主な機能:**

- カスタム原則定義
- 原則の検証と強制
- 例外の管理
- 原則の進化サポート

---

### [途中計画変更対応](./adaptive-replanning.md) 🆕

開発途中で仕様が変更されても、既存の進捗を無駄にせず、影響範囲を分析して適切に再計画。

**主な機能:**

- 差分ベースの再計画
- タスク依存関係の自動調整
- 変更履歴の自動記録
- 段階的移行のサポート
- コンフリクト検出

**使用例:**

```text
/ikak:replan "ユーザー認証にGoogle OAuth 2.0を追加"
```

---

## 機能の組み合わせ

これらの機能は独立して使用できますが、組み合わせることでさらに強力になります:

### 例: 安全な大規模リファクタリング

1. **柔軟な開発原則** で新しいアーキテクチャの原則を定義
2. **批判的対話** でリファクタリング計画を詳細化
3. **途中計画変更対応** で段階的な移行計画を作成
4. **GitHub 統合** で各フェーズを PR として管理
5. **三層記憶** で変更履歴と決定理由を記録
6. **デバッグ駆動修正** で問題が発生した場合に対応

### 例: 多言語チームでの開発

1. **多言語対応** で各メンバーの言語で仕様を記述
2. **段階的導入** で新メンバーのオンボーディングを支援
3. **批判的対話** で言語の壁を超えて意図を共有
4. **三層記憶** でチーム全体の知識を蓄積
